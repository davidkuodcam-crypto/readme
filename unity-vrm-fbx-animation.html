Unity VRM Animation Guide

VRM 1.0 動作套用、VRMA匯出與表情系統深度解析
完成度：0%

📝 課前準備

  • 軟體：Unity Editor (建議使用 2021.3 或 2022.3 LTS 版本)。
  • 核心套件：請至 UniVRM 官方 GitHub 下載主程式 VRM-0.xxx.x.unitypackage。
  • 匯出工具：請至 AnimationClipToVrma 下載專用轉換工具 AnimationClipToVrma_0.2.x.unitypackage。
  • 素材：一個 .vrm 虛擬角色檔案，以及一個 .fbx 動作檔案 (例如從 Mixamo 下載)。我們也會在任務四使用 Unity-Chan! Model 做為對照範例。

🚀 任務一：匯入 VRM 模型並套用 FBX 動作

將下載好的 核心套件 (VRM-0.xxx) 與 匯出工具 (AnimationClipToVrma) 依序拖曳進 Unity 下方的 Project 面板中安裝。接著，將準備好的 .vrm 檔案直接拖曳進 Project 面板，Unity 會自動生成角色的 Prefab，並將該 Prefab 拖曳到左側 Hierarchy 中。

將 .fbx 動作檔案拖曳進 Project 面板。點擊該 FBX 檔案，在右側的 Inspector 面板中切換到 Rig 標籤。

將 Animation Type 從 Generic 更改為 Humanoid，然後點擊右下角的 Apply。這能確保 FBX 的動作完美重定向 (Retargeting) 到 VRM 角色上。

在 Project 面板空白處點擊右鍵：Create -> Animator Controller，將其命名為 VRM_Anim。

點選 Hierarchy 中的 VRM 角色，在右側的 Animator 元件中，將剛剛建立的 VRM_Anim 拖曳到 Controller 欄位裡。

雙擊打開 VRM_Anim 進入 Animator 視窗。在 Project 面板中點擊 FBX 檔案旁邊的小三角形展開它，找到 Animation Clip (帶有三角形播放圖示的檔案)。

將該 Clip 拖曳進 Animator 視窗中，它會自動變成橘色的預設狀態 (Default State)。按下 Unity 頂部的 Play 播放鍵，角色就會開始動作了！

💾 任務二：將 FBX 動作匯出成 VRMA 檔案

傳統 FBX 動作需要透過外掛轉換才能變成標準的 VRMA 動作格式，方便在其他網頁或應用程式中重複使用。

確保您已經安裝了 AnimationClipToVrma 工具包。在 Unity 頂部選單點擊 VRM -> VRM Animation Exporter 開啟匯出專用視窗。

💡 其他方式：您也可以直接在 Project 面板中對著 Animation Clip 點擊「右鍵」，選擇 VRM -> Export VRM Animation 快速呼叫。

在開啟的 Exporter 視窗中，填入以下兩個關鍵欄位：

  • Animation Clip：拖入您展開 FBX 檔案後找到的三角形動作檔案。
  • Reference Model (參考模型)：拖入您在任務一匯入的 .vrm 角色 Prefab。這一步非常重要，轉換器需要參考標準的 VRM 模型來讀取正確的骨架比例。

確認視窗設定無誤後，點擊下方的 Export，設定檔名與存檔路徑，即可獲得標準的 .vrma 動作檔案。

🎭 任務三：VRM 表情系統 (Expressions) 深度解析

VRM 擁有標準化的表情系統，透過控制臉部網格的 BlendShape，我們可以在不同模型之間共用同一套表情程式碼。接下來我們將從基礎到進階，逐步掌握這套系統。

在 VRM 1.0 中，表情主要透過 ExpressionKey 來呼叫，並分為三大控制部位：

🙂 情緒 (Emotion)

控制眉毛、眼眶與整體臉部

  • ExpressionKey.Happy
  • ExpressionKey.Angry
  • ExpressionKey.Sad
  • ExpressionKey.Relaxed
  • ExpressionKey.Surprised
  • ExpressionKey.Neutral (恢復自然)

👄 唇型 (LipSync)

控制嘴巴開合與發音嘴型

  • ExpressionKey.Aa (ㄚ)
  • ExpressionKey.Ih (ㄧ)
  • ExpressionKey.Ou (ㄨ)
  • ExpressionKey.Ee (ㄝ)
  • ExpressionKey.Oh (ㄛ)

👀 眨眼 (Blink)

控制眼皮閉合

  • ExpressionKey.Blink (雙眼)
  • ExpressionKey.BlinkLeft (左眼)
  • ExpressionKey.BlinkRight (右眼)

建立一個 C# 腳本命名為 VRMBasicExpression，綁定到場景中的空物件上，並將 VRM 角色拖入 vrmInstance 欄位。這示範了如何透過權重 (0.0f 到 1.0f) 來開關表情。

using UnityEngine; using UniVRM10; public class VRMBasicExpression : MonoBehaviour { public Vrm10Instance vrmInstance; void Update() { if (vrmInstance == null) return; // 按下空白鍵時：將 Happy 權重設為 1.0 (完全開啟) if (Input.GetKeyDown(KeyCode.Space)) { vrmInstance.Runtime.Expression.SetWeight(ExpressionKey.Happy, 1.0f); } // 放開空白鍵時：將 Happy 權重設為 0.0 (關閉) if (Input.GetKeyUp(KeyCode.Space)) { vrmInstance.Runtime.Expression.SetWeight(ExpressionKey.Happy, 0.0f); } } }

有時候表情或說話不能只是 0 與 1 的切換，而是需要漸變。建立 VRMLipSyncExample 腳本，我們利用數學函式 Mathf.PingPong 讓角色連續發出「Aa(ㄚ)」的平滑嘴型，就像在呼吸或說話一樣。

using UnityEngine; using UniVRM10; public class VRMLipSyncExample : MonoBehaviour { public Vrm10Instance vrmInstance; public float talkingSpeed = 5.0f; // 嘴巴開合速度 void Update() { if (vrmInstance == null) return; // 計算平滑數值 (會在 0.0 到 1.0 之間來回彈跳) float weight = Mathf.PingPong(Time.time * talkingSpeed, 1.0f); // 將數值動態指派給 Aa(ㄚ) 嘴型 vrmInstance.Runtime.Expression.SetWeight(ExpressionKey.Aa, weight); } }

除了預設表情，許多模型師會在 VRM 中加入「星星眼 (StarEye)」、「貓耳下垂」等自訂表情。我們無法使用 ExpressionKey.Happy 這種預設語法，必須使用 CreateCustom 來動態呼叫。

using UnityEngine; using UniVRM10; public class VRMCustomExpression : MonoBehaviour { public Vrm10Instance vrmInstance; public string customExpressionName = "star_eye"; // 填入模型師設定的表情名稱 void Update() { if (vrmInstance == null) return; if (Input.GetKeyDown(KeyCode.Return)) // 按下 Enter 鍵 { // 建立自訂的 ExpressionKey 並賦予權重 1.0 ExpressionKey customKey = ExpressionKey.CreateCustom(customExpressionName); vrmInstance.Runtime.Expression.SetWeight(customKey, 1.0f); } } }

在開發 VRM 專案時，請務必遵守以下規範，避免產生 Bug：

不要直接修改 SkinnedMeshRenderer：
千萬不要用 Unity 原生的 SetBlendShapeWeight() 來改臉部網格。必須統一使用 vrmInstance.Runtime.Expression.SetWeight()。因為 VRM 內部有防止穿模的機制（例如設定了「大笑時覆蓋眨眼」），直接改網格會破壞這套保護機制。
權重的範圍：
傳入 SetWeight 的數值必須是 float 型態，且範圍建議嚴格限制在 0.0f 到 1.0f 之間。超過 1.0 雖然有時有效，但極易造成模型臉部破圖 (Mesh Distortion)。

🔍 任務四：FBX 模型與 VRM 的差異 (以 Unity-Chan 為例)

在 VRM 尚未普及之前，遊戲開發者通常直接使用 .fbx 格式的模型（例如著名的 Unity-Chan）。學習兩者的差異，能幫助你理解為什麼現代虛擬偶像與遊戲都傾向使用 VRM。

如果你從 Asset Store 下載 Unity-Chan! Model 並放入場景，你會發現她沒有 Vrm10Instance。要改變她的表情，你必須在她的臉部子物件 (例如 MTH_DEF 網格) 上找到 Skinned Mesh Renderer 元件。

最大痛點：Unity-Chan 的微笑可能叫 smile1，但換成另一個 FBX 角色可能叫 Face_Happy。因為缺乏標準，你每換一個 FBX 模型，程式碼就要全部重寫！

我們來看看控制傳統 FBX 模型 (如 Unity-Chan) 臉部表情的寫法。請注意這與 VRM 寫法的兩大差異：

  1. 必須直接取得 SkinnedMeshRenderer 元件。
  2. 權重數值 (Weight) 在 Unity 原生系統中是 0 到 100，而 VRM 是 0.0 到 1.0。
using UnityEngine; public class FBXExpressionExample : MonoBehaviour { // 需要手手動拖曳擁有 BlendShape 的「臉部網格」物件 public SkinnedMeshRenderer faceMeshRenderer; // FBX 的表情索引值 (可能要自己慢慢測試第幾個是笑臉) public int smileBlendShapeIndex = 0; void Update() { if (faceMeshRenderer == null) return; if (Input.GetKeyDown(KeyCode.Space)) { // Unity 原生的表情控制：(索引值, 權重 0~100) // 注意這裡輸入的是 100f，這與 VRM 的 1.0f 完全不同！ faceMeshRenderer.SetBlendShapeWeight(smileBlendShapeIndex, 100f); } if (Input.GetKeyUp(KeyCode.Space)) { // 恢復原狀，設為 0 faceMeshRenderer.SetBlendShapeWeight(smileBlendShapeIndex, 0f); } } }

透過比較，我們可以總結出 VRM (相對於 FBX) 在角色控制上的強大優勢：

  • 通用性：VRM 統一了 ExpressionKey.Happy，同一套程式碼套用在 A 角色和 B 角色上都會正確笑出來。
  • 防穿模機制：FBX 如果同時把「嘴巴張開」和「微笑」都調到 100，常常會導致牙齒穿出臉頰破圖；而 VRM 的設定檔允許防呆覆蓋。
  • 統一材質：除了表情，FBX 的材質球 (Material) 常常在不同電腦或專案中遺失或變成粉紅色；VRM 內建了 MToon 標準動漫材質，確保匯入即用。
